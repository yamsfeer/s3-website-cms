# 业务流程图

## 核心业务流程：访客浏览 → 提交线索 → 后台跟进

```
访客 --> 浏览官网首页 --> 查看服务详情 --> 提交在线留言
                                                    |
                                        +-----------+-----------+
                                        |                       |
                                        v                       v
                                    [提交成功]              [提交失败]
                                        |                       |
                                        v                       v
                                留言存入 messages 表      显示错误提示
                                is_read = 0              保留已填数据
                                        |
                                        v
                                后台留言列表显示新留言
                                （角标提示未读数量）
                                        |
                                        v
                                管理员查看留言内容
                                        |
                                        v
                                标记留言为已处理
                                is_read = 1
                                        |
                                        v
                                线下跟进客户
```

```
                     v      v      v
            [服务列表]  [新闻列表]  [在线留言]
            /services   /news      /contact
                |          |
                v          v
          [服务详情]   [新闻详情]
          /services/:id  /news/:id
                |          |
                v          v
          [立即咨询]   [返回列表]
                |
                v
          [在线留言 /contact]
```

## 后台 CMS 操作流程

```
[管理员登录] --> [仪表盘]
                    |
    +-------+-------+-------+-------+-------+
    |       |       |       |       |       |
    v       v       v       v       v       v
 [轮播图]  [服务]  [新闻]  [留言]  [设置]  [退出]
  管理     管理    管理    管理    管理    登录
    |       |       |       |       |
    v       v       v       v       v
  CRUD    CRUD    CRUD    查看    保存
  +上传    +上传    +富文本  +标记   配置
  +排序    +排序    +发布    已处理
```

## 内容发布流程

```
管理员 --> 进入后台 --> 新增/编辑内容
                            |
                +-----------+-----------+
                |                       |
                v                       v
          [新闻/服务/轮播图]      [站点设置]
                |                       |
                v                       v
        [填写表单+上传图片]     [修改配置字段]
                |                       |
                v                       v
        [选择是否启用/发布]     [点击保存]
                |                       |
                v                       v
        [保存到数据库]          [更新 settings 表]
                |                       |
                v                       v
        [前台自动展示最新内容]   [前台 SEO/联系信息更新]
```

## 文件上传流程

```
管理员 --> 选择图片文件 --> [格式校验]
                              |
                  +-----------+-----------+
                  |                       |
                  v                       v
              [通过]                  [不通过]
                  |                       |
                  v                       v
          [文件大小校验]          [提示"不支持的格式"]
                  |                       |
          +-------+-------+              |
          |               |              |
          v               v              |
      [≤5MB]          [>5MB]            |
          |               |              |
          v               v              |
  [生成时间戳文件名]  [提示"文件过大"]    |
          |                              |
          v                              v
  [保存到 /uploads/目录]            [重新选择]
          |
          v
  [返回可访问 URL]
          |
          v
  [写入数据库字段]
  (如 banners.image_url,
   services.icon,
   news.cover)
```

## 认证与鉴权流程

```
[请求到达]
      |
      v
[是否需要鉴权？]
      |
  +---+---+
  |       |
  v       v
[否]    [是]
  |       |
  v       v
[放行]  [检查 Authorization Header]
              |
      +-------+-------+
      |               |
      v               v
  [有 Token]      [无 Token]
      |               |
      v               v
  [验证 JWT 签名]  [返回 401]
      |
  +---+---+
  |       |
  v       v
[有效]  [无效/过期]
  |       |
  v       v
[放行]  [返回 401]
```

## 部署架构流程

```
[Nginx :80]
      |
      +-- /              --> 前台静态文件 (dist/frontend/)
      |
      +-- /admin         --> 后台 SPA (dist/admin/)
      |
      +-- /api           --> 反向代理到 Express :3000
      |
      +-- /uploads       --> 静态文件托管 (uploads/)

[Express :3000]
      |
      +-- /api/auth      --> 认证路由
      +-- /api/messages  --> 留言路由
      +-- /api/services  --> 服务路由
      +-- /api/news      --> 新闻路由
      +-- /api/banners   --> 轮播图路由
      +-- /api/upload    --> 上传路由
      +-- /api/settings  --> 设置路由
      +-- /api/dashboard --> 仪表盘路由

[SQLite]
      |
      +-- data/app.db    --> 数据库文件
```
